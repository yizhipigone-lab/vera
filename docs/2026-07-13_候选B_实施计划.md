# 候选 B 实施计划：35 脚本收口到 BacktestRunner

> 日期：2026-07-13。依赖：候选 A 阶段 1（run_cached 加厚前门，已完成 980b04f）+ 阶段 1.5（3 脚本收编，已完成 ca8bc6e）。

## Context（为什么做）

VERA 根目录 90 个 .py 脚本（11 天内从 53 涨到 90），其中 35 个调 `run_cached`，各自重写"跑一次回测"序列：
- `ENGINE_CFG` 28 处手写（无 loader，两个资本金派系 100w vs 20w）
- `STOP_CONFIG` 27 处三来源（8 手写 + 14 loader + 2 defaults.get，-0.08 vs -0.12 两派系）
- `sys.path.insert` 74 处 6 种写法变体
- TDX 函数 lambda 字典 5 脚本各写一份（25/33/23/18/3 个函数，core/formula_runner 有正式实现却不用）
- `BacktestEngine(ENGINE_CFG)` + `run_cached(10参手写排)` 30+ 处

**局部性失守**：改 `run_cached` 签名要同步改 35 脚本；其中 2 个还绕过 run_cached 直调私有（阶段 1.5 已收编）。候选 A 已把 run_cached 深化成深接口，候选 B 抽 `BacktestRunner` 把"跑一次回测"的序列收口，35 脚本变成 1 行调用。

## 设计：BacktestRunner 深模块

**接口草案**（design-it-twice 选"厚前门"方向，与 run_cached 一致）：
```python
class BacktestRunner:
    def __init__(self, config: dict | str = None):
        """config: engine_cfg + stop_config + time_range + universe + formula。
        str 时视为 strategy yaml 路径，从 ConfigLoader 加载。None 用 default.yaml。"""

    def run(self, strategy: dict, *, skip_sm=True, filter_limit_up=True,
            close=None, entries=None, **caps) -> dict:
        """跑一次回测: 取价→选股→公式→engine.run_cached→metrics。
        close/entries 提供时跳过取价/选股（批量优化场景）。"""
```

35 脚本从 10 行拼装 → `BacktestRunner(cfg).run(strategy)`。

**接缝背后放什么**（从调用方/三层收进 Runner 内部）：
- TDX 连接（TdxConnector.ensure_connected）
- 取价（DataFetcher.get_kline/get_close_price）
- 选股（StockSelector + filter_stocks）
- 公式执行（FormulaRunner，删 5 处 lambda 字典）
- ENGINE_CFG/STOP_CONFIG 构造（从 ConfigLoader 读，删 28+27 处手写——顺势做候选 C）
- BacktestEngine 构造 + run_cached 调用
- metrics + equity_curve

## 实施步骤（分 5 阶段，每阶段可独立交付 + parity 验证）

1. **建 backtest/runner.py**（Runner 深模块）：封装取价/选股/公式/engine/run_cached/metrics。TDD：行为测试（合成数据 + parity vs 现有 run_cached 直调）。
2. **config 门面（候选 C 顺势）**：ConfigLoader 吞下 backtest + stop_loss + engine_cfg 三段，删 28 处 ENGINE_CFG 手写 + 8 处 STOP_CONFIG 手写。缺失即 raise。
3. **迁移 35 脚本（分批）**：
   - backtest_ 10 个（最简单，先做）
   - batch_ 11 个
   - optimize_ 16 个（最大族）
   - run_ 7 + analyze_ 3
   每个改成 `BacktestRunner(cfg).run(strategy)`，用 golden output 对比。
4. **删 5 处 TDX lambda 字典**：batch_b_formulas/batch_eval_v3/batch_final/batch_eval/batch_fixed 改用 core/formula_runner。
5. **统一 sys.path.insert**：74 处 → 0（用 `pip install -e .` 或 conftest.py 自动加路径）。

## 风险与对策

| 风险 | 对策 |
|---|---|
| 35 脚本变体多（有的不选股/自定义取价/多策略对比） | Runner 用 config 开关 + close/entries 预提供参数支持批量优化场景 |
| 迁移面大，实盘脆弱期 | 分批（5 阶段），每批 parity + golden output 验证；回滚 = git revert 单批 |
| optimize_strategies 的 stop_cfg 默认值坑（阶段 1.5 已发现） | config 门面强制完整 stop_config，删散落默认值 |
| 3 生产脚本（阶段 1.5 收编的）需 TDX 验证 | 候选 B 迁移时一并跑 golden output |

## 验证
