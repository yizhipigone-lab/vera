## 第一部分：三角度审计报告

> 对上一版"思路"的审计，找出 15 个问题，按角色分组。

### 一、架构设计师视角（5 项）

#### A1. 前后端数据契约不统一 ❗
- **问题**：`StrategyConfig.sectors` 是 `str`（逗号分隔），`_config_to_yaml_dict` 转成 `list`，`selector` 读 `list`，前端发什么没明确。三段类型不一致易出 bug。
- **改进**：统一契约——前端发**逗号分隔字符串**（跟 `ladder_levels` 同风格），后端 `StrategyConfig.sectors: str`，`_config_to_yaml_dict` 解析成 `list[str]`，`selector` 读 `list`。全程类型清晰。

#### A2. 板块列表 API 缺乏容错 ❗
- **问题**：`/api/sectors` 调 TDX 拉板块列表，TDX 断连时返回什么没设计。前端会拿到 500 错误，checkbox 区域空白，用户不知道怎么回事。
- **改进**：API 加 try/except，TDX 断连时返回 `{success: False, error: "TDX 未连接", sectors: []}`。前端显示"板块列表加载失败，请检查通达信客户端"+ 重试按钮。

#### A3. 板块成份股没缓存 ❗
- **问题**：用户重复跑同一组板块回测，每次都重新调 `get_stock_list_in_sector`，浪费 1-3s。
- **改进**：`DataFetcher` 加进程级缓存 `_SECTOR_STOCKS_CACHE: Dict[str, List[str]]`，板块成份股变化极少，缓存命中后 <10ms。提供 `clear_sector_cache()` 供手动刷新。

#### A4. 多板块串行调用的用户体验 ❗
- **问题**：选 10 个板块串行调 TDX ~3s，前端 `runPipeline` 进度条只显示"执行选股"，用户不知道在等什么。
- **改进**：后端 `selector.resolve_universe()` 加日志 `拉取板块成份股 [i/N]: 半导体 (180 只)`；前端进度提示细化（可选，后端日志够用）。

#### A5. 板块 code vs name 选择 ❗
- **问题**：中文名理论上可能重名（虽然 TDX 板块名唯一，但传输用中文有编码风险）。前端 checkbox `value` 用什么没定。
- **改进**：前端 checkbox `value` 用**板块代码**（`881319.SH`，稳定唯一），`label` 用中文名。传后端用代码。后端 `get_stock_list_in_sector` 实测支持代码入参。这样最稳，避免中文编码问题。

### 二、功能设计师视角（5 项）

#### F1. 128 个 checkbox 的可用性 ⚠️
- **问题**：128 个 checkbox 直接堆出来太长，用户找"半导体"要滚半天。
- **改进**：搜索框**实时过滤**（`oninput` 事件，不区分大小写匹配中文名）+ 可滚动容器（`max-height: 200px`）。128 个 DOM 节点浏览器能扛，不需要虚拟滚动。

#### F2. 跟"股票池下拉框"的关系未明确 ❗
- **问题**：选了行业板块后，上方"股票池"下拉框（沪深A股/沪深300...）还有效吗？我之前设计说"板块成份股为基池，下拉框被忽略"，但 UI 上没体现，用户会困惑。
- **改进**：选了任何板块后，"股票池"下拉框**变灰禁用** + tooltip "已选行业板块，下拉框被忽略"。清空板块后下拉框恢复。

#### F3. 跟 ETF 开关的叠加关系 UI 提示 ⚠️
- **问题**：用户可能不清楚"选了板块 + 勾包含ETF" = 板块 + ETF 池叠加。
- **改进**：板块标题旁加说明文字："勾选后将与上方 ETF 开关叠加（板块成份股 ∪ ETF）"。

#### F4. 已选状态反馈 ⚠️
- **问题**：勾了几个板块，用户要看清自己选了啥。
- **改进**：板块列表下方实时显示"已选: 半导体, 消费电子 (2 个)"，随勾选动态更新。

#### F5. 全选/清空按钮 ⚠️
- **问题**：128 个全选意义不大（等于全 A 股），但清空有用。
- **改进**：只保留"清空"按钮，去掉"全选"。搜索框 + 清空足够。

### 三、项目质量设计师视角（5 项）

#### Q1. 测试方案缺失 ❗
- **问题**：`selector.sectors` 分支依赖 TDX，怎么写单元测试？
- **改进**：测试用 `monkeypatch` mock `DataFetcher.get_sector_stocks`，返回固定成份股列表，不依赖 TDX。测试用例：
  - 单板块 → 成份股正确
  - 多板块 → 并集去重
  - 板块 + include_etf → 叠加 ETF
  - 板块 + etf_only → 只 ETF（板块被忽略）
  - 空板块列表 → 走原 A 股路径

#### Q2. 验收标准不量化 ❗
- **问题**："跑一次回测看 trades 代码分布"太模糊，无法自动化验证。
- **改进**：量化验收——勾选"半导体"板块跑回测，断言所有 trades 的 `stock_code` ∈ 半导体 180 只成份股集合。

#### Q3. 工作量饱满性 — 阶段 2 太粗 ⚠️
- **问题**：阶段 2（前端）只写"checkbox 列表 + 搜索"，实际涉及 HTML/CSS/JS 三块，任务粒度太粗。
- **改进**：拆细为 2a（HTML 结构 + CSS）、2b（JS 渲染 + 搜索过滤）、2c（JS localStorage + runPipeline 传参 + 下拉框联动禁用）。

#### Q4. 回滚方案不明确 ⚠️
- **问题**：出了问题怎么回滚没写。
- **改进**：`sectors` 默认空数组 → 不勾任何板块 → 行为与现状完全一致。出问题只需前端清空勾选 / yaml 删 sectors 字段。

#### Q5. 边界 case 清单缺失 ❗
- **问题**：板块列表为空、板块名错误、成份股为空、TDX 断连等边界没列。
- **改进**：列边界 case 表 + 处理方式（见下文 3.4 节）。

## 第二部分：正式实施计划书

### 1. 背景与目标

**背景**：VERA 当前选股池只能是"全市场 A 股 / 某指数成份股 / ETF"。用户需要**按行业板块选股**（如只在"半导体"板块里跑选股公式），且支持**多板块叠加** + **与 ETF 开关叠加**。

**目标**：
1. 用户可多选 128 个细分行业板块（list_type=11）作为选股池
2. 板块池可与 ETF 开关叠加（板块 ∪ ETF）
3. 数据来源全走通达信（板块列表 + 成份股都从 TDX 拉）
4. 向后兼容（不选板块 = 现状不变）

### 2. 需求范围

#### 2.1 In Scope（本次做）
- 后端：`DataFetcher.get_sector_stocks` + `selector` sectors 分支 + `server.py` 透传 + `/api/sectors` 端点
- 前端：可搜索的 checkbox 列表 + 清空按钮 + 已选反馈 + 下拉框联动禁用
- 测试：selector sectors 分支单测（mock TDX）
- 配置：yaml 加 `sectors` 字段

#### 2.2 Out of Scope（本次不做）
- 板块成份股的实时刷新（用进程级缓存，手动 clear）
- 虚拟滚动（128 个不需要）
- 板块成份股交集模式（本次只做并集）
- 自定义板块（list_type=0 自选股）

### 3. 架构设计

#### 3.1 前后端数据流

```
[页面加载]
  前端 → GET /api/sectors → 后端 → tq.get_stock_list('11', list_type=1)
                                      → 返回 128 个 {code, name}
                                      → 进程级缓存 _SECTOR_CACHE
  前端渲染 checkbox 列表 (value=code, label=name)

[用户勾选 + 跑回测]
  前端 runPipeline():
    sectors = "881319.SH,881326.SH"  // 逗号分隔代码
    POST /api/run { sectors: "881319.SH,881326.SH", include_etf: false, ... }
  
  后端 _config_to_yaml_dict():
    universe.sectors = ["881319.SH", "881326.SH"]  // 解析成 list
  
  后端 selector.resolve_universe():
    for code in sectors:
      stocks += DataFetcher.get_sector_stocks(code)  // 命中缓存 <10ms
    stocks = list(set(stocks))  // 并集去重
    if include_etf:
      stocks = list(set(stocks) | set(ETF池))
    → 选股公式照跑
```

#### 5.2 关键代码片段

**DataFetcher 加板块方法**：
```python
_SECTOR_CACHE: List[dict] = []          # 128 个板块列表
_SECTOR_STOCKS_CACHE: Dict[str, List[str]] = {}  # code → 成份股

@classmethod
def get_sector_list(cls) -> List[dict]:
    """获取 128 个细分行业板块 (list_type=11), 带进程级缓存"""
    if cls._SECTOR_CACHE:
        return cls._SECTOR_CACHE
    cls._ensure_ready()
    from tqcenter import tq
    raw = tq.get_stock_list('11', list_type=1)
    cls._SECTOR_CACHE = [
        {"code": s["Code"], "name": s["Name"].strip()}
        for s in raw if isinstance(s, dict)
    ]
    return cls._SECTOR_CACHE

@classmethod
def get_sector_stocks(cls, sector_code: str) -> List[str]:
    """拉板块成份股 (支持板块代码或中文名), 带进程级缓存"""
    if sector_code in cls._SECTOR_STOCKS_CACHE:
        return cls._SECTOR_STOCKS_CACHE[sector_code]
    cls._ensure_ready()
    from tqcenter import tq
    raw = tq.get_stock_list_in_sector(sector_code, list_type=0)
    stocks = [s if isinstance(s, str) else s.get("Code","") for s in raw]
    stocks = [s for s in stocks if s]
    cls._SECTOR_STOCKS_CACHE[sector_code] = stocks
    return stocks

@classmethod
def clear_sector_cache(cls):
    """清空板块缓存 (板块成份股更新时手动调)"""
    cls._SECTOR_CACHE.clear()
    cls._SECTOR_STOCKS_CACHE.clear()
```

**selector resolve_universe sectors 分支**：
```python
sectors = u.get("sectors", [])
if sectors:
    # 选了行业板块
    stocks = []
    for i, code in enumerate(sectors):
        sector_stocks = DataFetcher.get_sector_stocks(code)
        logger.info(f"拉取板块成份股 [{i+1}/{len(sectors)}]: {code} ({len(sector_stocks)} 只)")
        stocks.extend(sector_stocks)
    stocks = list(set(stocks))  # 并集去重
    logger.info(f"板块并集: {len(stocks)} 只 (来自 {len(sectors)} 个板块)")
    # ETF 叠加
    if include_etf:
        etf_stocks = DataFetcher.get_stock_universe("31")
        stocks = list(set(stocks) | set(etf_stocks))
        logger.info(f"叠加 ETF: {len(stocks)} 只")
elif etf_only:
    stocks = DataFetcher.get_stock_universe("31")
else:
    list_type = UNIVERSE_TYPE_MAP.get(str(utype), utype)
    stocks = DataFetcher.get_stock_universe(list_type)
    if include_etf:
        stocks = list(set(stocks) | set(DataFetcher.get_stock_universe("31")))
```

| 错误 | 处理 |
|---|---|
| TDX 断连 | `/api/sectors` 返回 success=false；前端显示重试按钮 |
| 板块代码失效 | `get_sector_stocks` 返回空，日志 warning，跳过 |
| 板块列表为空 | sectors 区域显示"暂无板块数据" |
| 用户没勾任何板块 | 走原 A 股路径，无影响 |

#### 8.2 回滚方案

- **最简回滚**：前端清空所有板块勾选 → sectors="" → 行为与现状完全一致
- **代码回滚**：`git revert` 涉及的 7 个文件，无数据库/配置迁移

#### 8.3 向后兼容

- `sectors` 默认空 → 老配置 / 老脚本不受影响
- 老策略 yaml 没有 `sectors` 字段也能跑（`u.get("sectors", [])` 返回空）

### 9. 风险与对策

| 风险 | 概率 | 影响 | 对策 |
|---|---|---|---|
| TDX 板块接口不稳定 | 低 | 板块列表拉不到 | 进程级缓存 + 错误提示 + 重试 |
| 多板块串行调用慢 | 中 | 选股阶段 1-3s | 日志进度提示；串行保守（不并行，避免 TDX 单连接问题） |
| 板块成份股缓存过期 | 低 | 用了旧成份股 | 提供 `clear_sector_cache()` 手动刷新；板块成份股变化极少 |
| 前端 128 个 checkbox 性能 | 低 | 渲染略慢 | 128 个 DOM 节点可接受；搜索过滤用 display:none |
| 中文字符编码 | 低 | 板块名乱码 | 前后端都用 UTF-8；checkbox value 用代码不用中文 |
