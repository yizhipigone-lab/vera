## 🤔 额外隐患（穷举）

1. **a1173c5 在"翻默认"那一刻会从"无害"变"危险"**：use_cache=False 时所有 HIGH/CRITICAL 都是休眠的。一旦翻 True 或接 engine，C1+H1+H2+H3 同时激活。修复必须发生在翻默认之前，否则是把一堆休眠雷接进生产。
2. **parquet 是单点**：`_read_parquet` 对损坏文件无自愈（L6），一次 bit rot 或手工删改会让该 stock 永久 fetch_fail 直到手工删文件。无 checksum/隔离。
3. **日历 + 停牌 + 分红三者都是"时间敏感数据源"，但缓存对三者的时间语义处理不一致**：日历不刷新（M4）、停牌误判（H1）、分红 staleness 未做（H2）——根因都是"缓存当静态数据存，实际是时变数据"。这三者需要一个统一的"时变数据失效"框架，而不是各自打补丁。
4. **fake 世界太干净是系统性盲区**：10 个缓存测试全用"工作日日历 + 无停牌 + 无分红 + 单股"的 fake，所以 H1/H2/H4/多股缺口这些只能在脏真实世界暴露的问题全部漏网。测试质量决定了这些 bug 只能被审计（而非测试）抓住。
