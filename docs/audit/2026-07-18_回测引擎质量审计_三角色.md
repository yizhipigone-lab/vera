## 2. 主会话复核过的重点发现

### ✅ 已复核 CONFIRMED — 建议立即修(都是小改动)

**F1. 非二值信号矩阵静默前视 [MEDIUM, 昨日新引入 1b1335a]**
- [backtest/loop/loop.py:80-82](backtest/loop/loop.py#L80): `np.nonzero` 按**个数**切段, `entry_np.sum(axis=1)` 按**取值**求和。bool 矩阵两者相等(当前全部调用方都是 bool, 安全); 但 legacy 只判 truthy, 接受任意非零值。
- **主会话探针实测**: int 型 `entry=[[2,0],[0,2],[2,2]]` → legacy 在 bar0/bar1 正确买入, 新路径把 bar2 的信号挪到 bar0 成交(**前视**), 无任何报错。float 型则响亮 TypeError(可接受)。
- 修法(一行): `cuts = np.cumsum(np.count_nonzero(entry_np, axis=1))[:-1]` — 恢复与 legacy 完全一致的 truthy 语义。

**F2. BacktestLoop 实例复用带脏状态 [HIGH, 预先存在]**
- [backtest/loop/loop.py:73](backtest/loop/loop.py#L73): `run()` 不重置 PositionBook; [backtest/loop/entry.py:35](backtest/loop/entry.py#L35) `skipped_signal_count` 只在 `__init__` 归零。
- **主会话复核**: 持仓全部平掉的场景 run1==run2(所以 54 组 parity 抓不到); 但**期末有未平仓持仓时**, 第二轮带着上一轮的仓位跑, 资金数字静默错乱(架构 agent 实测 102,948 vs 141,976)。
- 生产路径当前安全([backtest/engine.py:82-96](backtest/engine.py#L82) 每次新建 loop), 属潜伏陷阱。
- 修法: `run()` 开头加 `_has_run` 守卫, 第二次调用 fail-fast(强迫每次新建, 语义最诚实)。

**F3. ExitDispatcher 快照 vs 公开可变 dict [MEDIUM, 昨日新引入 aac40ab]**
- [backtest/loop/exit_engine.py:66-77](backtest/loop/exit_engine.py#L66): `self.strategies` 保留公开可变, 但 `_order/_tail` 等在 `__init__` 一次性快照。构造后增删 strategies → 静默不生效/继续触发, 不报错。
- 修法: docstring 写明"构造后不可变, 改动请重建"(零成本), 或 MappingProxyType 只读暴露。

### ✅ 已复核 CONFIRMED — 排期修

**F4. 5m 窗口块零测试 [CRITICAL-测试, 预先存在]**
- [backtest/engine.py:758-824](backtest/engine.py#L758) `get_kline_windowed` 调用 + win_td 自动加长 + `last_tradable_idx` 窗口内重算 — 主会话 grep 验证 tests/ **零命中**。这是 5m 实盘活路径, 正是 002008 bug 的栖息地, parity 测试全部绕开此块(直传 tradable_np)。
