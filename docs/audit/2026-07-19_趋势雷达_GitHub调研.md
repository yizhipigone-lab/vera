> **日期**:2026-07-19
> **调研方法**:24 次 `gh search repos --sort stars` + 3 次 WebSearch,中英文都搜

> **补充**:last30days-skill(mvanhorn,52.6k)—— 用户指定考察

## 核心结论先行

- "多 Agent + LLM + A股另类数据"路径,**有现成旗舰可直接 fork**:`TradingAgents-astock`(2480)+ `aiagents-stock`(1711),叠加几乎覆盖全部诉求。
- "板块轮动趋势雷达"的**国际事实标准是 RRG(相对轮动图)**,Python 实现成熟,但**没有"A股概念板块 + 30天热度"成品**,要自研。
- 真正缺、必须自研的 5 件:**3 小时调度器集成**、**选股 × 趋势碰撞 join**、**LLM 10天趋势预测 + 回测验证**、**概念板块 30 天热度累积**、**关键词命中引擎**。

## 推荐组合(三档)

### 档位一:MVP(2 周)
fork `aiagents-stock` 或 `TradingAgents-astock`(已打通 A股另类数据+板块轮动预警+多Agent+告警),做三件事:
1. 数据层换 akshare + tushare 双源
2. 趋势排名叠 RRGPy 的 RRG 算法
3. 自研选股 × 趋势碰撞 join

### 档位二:平衡(4-6 周)
数据(akshare+tushare+adata+Ashare)+ 新闻(finance-news-aggregator+EastMoney_Crawler+自研)+ 趋势(RRGPy+看板)+ 舆情(FinGPT+FinRobot)+ 决策(TradingAgents-astock)+ LLM预测(自研prompt+Event-Driven回测)+ 调度(PanWatch+Sequoia-X)+ 回测(qlib+VERA 引擎)

### 档位三:豪华(8-12 周,生产级)
数据层接 OpenBB,qlib 做趋势因子库 + RD-Agent 自动挖因子,FinRL 做 RL 预测,TradingAgents 原版+中文版机构级多 Agent。

## 必须自研的 5 个缺口

1. **3 小时调度器 + 状态机**(PanWatch 调度最接近,可扒)
2. **选股信号 × 趋势排名 碰撞 join**(VERA 独有业务接缝,无通用开源)
3. **LLM "未来10天趋势" prompt + 评分卡回测**(无可信开源,参考 Event-Driven)
4. **A股概念板块 30 天热度时间序列**(akshare 有当日成分,30天曲线需每日落库累积)
5. **财经新闻+政策+产经关键词命中引擎**(自研轻量规则即可,不需 LLM)

## 与 VERA 现有架构的接缝

- 趋势雷达 = **独立新模块**,新建 `trend_radar/` 顶层目录,不塞进 `backtest/engine.py`(遵守"不破坏现有")
- 碰撞接缝在 `server.py` 的 `Pipeline.run` 之后(`PipelineResult` frozen dataclass 加字段),不动现有路径
- 复用 `core/dividend_type.py` 复权口径统一(别再分裂口径)
- 本地部署:FinGPT/FinRobot 支持本地模型,akshare/tushare 纯接口,符合"本地优先+策略机密"
