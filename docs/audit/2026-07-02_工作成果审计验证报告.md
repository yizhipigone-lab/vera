## 2. 发现的逻辑漏洞与错误

### 🔴 漏洞 1（HIGH）：T+1 买入时 open 价格被 ffill 污染——停牌日可能用虚假开盘价成交

**位置**：`engine.py:459`

```python
open_np = open_df.reindex(index=idx, columns=cols).ffill().values.astype(np.float64)
```

**问题**：

close 用 `close_raw.ffill()` 做 mark-to-market 是合理的（需要知道停牌日市值）。但 **open 也做了 ffill 就不对了**——open 的唯一用途是"T+1 次日开盘买入"的成交价。如果某只股停牌，TDX 返回的 open 是 NaN，ffill 会把 NaN 填成上一个交易日的开盘价。

然后 T+1 买入逻辑 `engine.py:295-296` 检查 `np.isnan(bp)` 时，ffill 已把 NaN 清掉了，就会误判为"有开盘价、可以成交"。

**后果**：信号日次日如果停牌，本应跳过该信号，但实际会用**上一天的开盘价**买入——这是虚假成交价，属于轻度前视偏差。

**复现场景**：

1. Day i-1：信号触发，`entry_np[i-1, ci] = True`
2. Day i：该股停牌，TDX 返回 open=NaN
3. `open_df.ffill()` → open[i, ci] = open[i-1, ci]（前一天开盘价）
4. T+1 买入逻辑：`bp = open_np[i, ci]` → 非 NaN → 成交
5. 结果：以一个不存在的价格买入了一只停牌股

**修复建议**：

open 不应 ffill，应保留 NaN 让 `np.isnan(bp)` 自然拦截：

```python
# 修复前
open_np = open_df.reindex(index=idx, columns=cols).ffill().values.astype(np.float64)

# 修复后
open_np = open_df.reindex(index=idx, columns=cols).values.astype(np.float64)  # 保留 NaN
```

**影响评估**：只影响 run() 路径（run_cached 不传 open_np），且仅当"信号日次日恰好停牌"时触发。频率低但逻辑上不可接受。

### 🟡 漏洞 3（MEDIUM）：CORS allow_origins 包含 "null" 削弱安全防护

**位置**：`server.py:31`

```python
allow_origins=["http://127.0.0.1:8080", "http://localhost:8080", "null"]
```

**问题**：`"null"` 是浏览器在 `file://` 协议、sandboxed iframe、data: URL 下发送的 origin 值。包含它意味着任何本地 HTML 文件或恶意 iframe 都能向 VERA server 发跨域请求。这与"CORS 收紧"的初衷矛盾。

**修复建议**：移除 `"null"`。如确需本地 HTML 访问，`http://localhost:8080` 已覆盖。

**位置**：`web/index.html:215` 和 `web/index.html:226`

| 移动止损回撤 | **5%** | 回撤 **3%** 触发 | ❌ 差 2pp |

**问题**：报告声称"默认值与 yaml 一致"，engine/server 后端确实一致了。但**前端 HTML 硬编码的默认显示值还是旧的**。用户打开页面看到的止损参数和实际跑的差了好几个百分点。

**这是 P0-2 修复的遗漏——修了后端没修前端。**

**修复建议**：将 `index.html:215` 改为 `亏损达到 <b>12%</b>`，`index.html:226` 改为 `盈利 <b>8%</b> 激活后，回撤 <b>5%</b> 触发`。
