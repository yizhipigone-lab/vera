## 一、Context（为什么做这件事）

VERA 当前的止损止盈系统（`backtest/stop_manager.py` + `backtest/engine.py:_simulate_core_v3`）有 6 个机制：

1. 成本止损（Low 跌破止损线）
2. 移动止损（High 激活后回撤）
3. 阶梯止盈（多档分批）
4. 时间止盈（持仓 N 天）
5. 条件时间止盈（持仓 N 天且盈利≥X%）
6. 首日未达标（买入次日涨幅未达标）

**全部基于 OHLC 算出来**，**完全不知道用户在 TDX 写的公式里有什么信号**。

如果某个公式已经写了"放量跌破 20 日均线就卖"，StopManager 还在傻乎乎地按 -8% 止损——**两边可能打架**，或者公式想表达的卖出意图被压价规则吞掉。

**本次需求**：在止损止盈体系里**新增一个公式卖出机制**——每天检查持仓股在某条 TDX 公式下是否有信号，有就按配置的比例卖出。**不管盈亏**（有信号就卖），**最高优先级**（早于成本止损），**公式名可填**，**卖出比例可填**，**可设置是否生效**。

**用法**：回测验证——加入"公式卖出"前后，对比回测曲线变化，验证机制有效性。

### 阶段 4：TDX 集成 + 缓存接入 ✅

**修改** `backtest/engine.py:run()` —— 在调 `_simulate_core_v3` 之前插入预计算代码（~80 行）

```python
formula_exit_np = None
formula_exit_ratio = 1.0
fs_cfg = stop.get("formula_sell", {})
if fs_cfg.get("enabled", False):
    formula_name = str(fs_cfg.get("formula_name", "")).strip()
    formula_arg = str(fs_cfg.get("formula_arg", ""))
    formula_exit_ratio = float(fs_cfg.get("sell_ratio", 1.0))
    if formula_name and start_time and end_time:
        try:
            # 1. 计算缓存键
            key = cache_key(formula_name, formula_arg, tuple(cols),
                            start_time, end_time, period=self.period)
            # 2. 查缓存
            cached = load_cached_formula_exit(key)
            if cached is not None:
                formula_exit_np = cached.matrix
                logger.info("formula_sell: 命中缓存 [%s] (信号=%d)",
                            formula_name, int(formula_exit_np.sum()))
            else:
                # 3. 调 TDX (唯一一次 RPC)
                sig_df = FormulaRunner.run_stock_selection_with_dates(...)
                # 4. 构造矩阵 + 写缓存
                formula_exit_np = build_formula_exit_matrix(sig_df, close.index, close.columns)
                save_cached_formula_exit(key, FormulaExitResult(matrix=..., meta={...}))
        except Exception as e:
            logger.error("formula_sell 构造失败, 回退禁用: %s", e)
            formula_exit_np = None
```

**调 `_simulate_core_v3` 时严格 keyword 传参**：

```python
equity_arr, raw_trades = _simulate_core_v3(
    ..., 
    formula_exit_np=formula_exit_np,
    formula_exit_ratio=formula_exit_ratio,
    formula_exit_lag_bars=1,
)
```

**关键设计决策**：

| 决策 | 选择 | 理由 |
|---|---|---|
| RPC 调用位置 | `engine.run()` 入口预计算 | 避免 hot loop 内 RPC（TDX 单连接线程不安全） |
| 缓存 | 24h TTL + SHA-256 key | 重复跑同一公式+股票池直接命中缓存 |
| 失败兜底 | try/except 包整个 TDX 调用 | 任何异常都不会让回测运行崩，回退到 `formula_exit_np=None` |
| `run_cached()` 路径 | 不调 TDX | 批跑优化场景，由调用方显式传 `formula_exit_np=` keyword |

**验收**：模块导入 + 集成自检全部通过 ✅

## 六、关键设计决策回顾

| 决策点 | 选择 | 替代方案 | 理由 |
|---|---|---|---|
| reason code | 12 | 11（已占用） | 不冲突退市（11） |
| 触发优先级 | cost_stop 之前 | cost_stop 之后 | 用户要求"最高优先级" |
| 触发条件 | 仅 matrix[True] | matrix + 盈亏判断 | 用户明确"不管盈亏" |
| sell_price | Close（fallthrough else） | 自定义公式卖价 | 公式只决定卖不卖，价格用市价 |
| TDX 调用 | run() 入口预计算 | 主循环内 RPC | 单连接线程不安全，性能差 |
| 缓存 TTL | 24h | 永久 / 1h | 平衡"新信号及时"与"避免重复 RPC" |
| 默认 enabled | false | true | 安全优先，防止误触发 |
| 部分卖出模式 | 复用 ladder 模式 | 新 partial-sell 函数 | 与现有约定一致 |
| 缓存路径 | `~/.cache/vera/formula_exit/` | 工程内 `output/` | 跨项目统一，OS 缓存约定 |
| 回退机制 | try/except 包整个 TDX 调用 | 分段 try/except | 一处失败全回退到 disabled |

## 七、风险与边界

### 已识别风险

1. **TDX 客户端未启动**：`FormulaRunner` 会抛 `RuntimeError` → 后端 try/except 兜底 → 写 warning log → 回退 `formula_exit_np=None` → 回测与改造前完全一致 ✅
2. **公式名拼写错误**：`tq.formula_process_mul_xg` 返回空 → `build_formula_exit_matrix` 返回全 False 矩阵 → 无信号触发 → 用户看到日志 `信号=0` ✅
3. **卖出比例填 150%**：yaml 配置层未校验（前端 JS 加了 `[0%, 100%]` 钳位，待前端实施后生效）；后端若收到 `sell_ratio > 1.0`，partial-sell 路径 `sell_sh = int(total_sh * 1.5) = total_sh`，自然落到全卖分支 ✅
4. **位置参数契约**：`test_engine_run_path.py:44` 锁死 `_simulate_core_v3` 位置参数 ≤ 22，本次新增参数全部 keyword-only，**契约未被破坏** ✅
5. **T+1 保护**：matrix[5,0]=True 不会在 bar 6 触发（被入场日 continue 跳过）；正确用法是 matrix[6,0]=True → bar 7 触发 ✅

### 已知局限

1. **性能基线**：1000 只 × 3 年 ≈ 5 秒（基于 a2_step1_v2_batch100.log 实测）。缓存命中后 < 100ms。
2. **多公式支持**：当前 `formula_sell.formula_name` 只接受单一公式。如果要"多公式任意一个命中即卖"，需要扩展为 list。
3. **公式参数**：当前 `formula_arg` 字段预留，但前端未暴露 UI；后端已透传。
4. **A股 T+1 规则**：当日买入次日才可卖。公式信号日 = TDX 返回的 date → 主循环 i - 1 = 信号日 → 触发 bar = 信号日 + 1。这与 entry 的 T+1 规则一致。

### 不在本次范围

1. **实盘路径（`server.py` / `web/` / `run_live_*`）**：本轮只覆盖回测验证，实盘接入留到下一轮。
2. **回测报告 UI 改造**：trades 表里已经会显示 "formula_sell" 字符串，pie chart 自动适配。
3. **TDX 客户端自动启动/检测**：假设用户已手动开好。
4. **前端 UI 接入**：plan 已写（详见 `C:\Users\liuziheng\.claude\plans\steady-orbiting-whale.md`），待批准后实施。
