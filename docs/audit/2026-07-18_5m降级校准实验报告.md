# 5m 降级偏差校准实验报告

> 2026-07-18 · 计划书 Phase C · 工具: `tools/calibrate_degrade_5m.py` · 数据: 真实 TDX + KlineCache

## 结论 (先说)

1. **复权口径一致时, 降级无损** — 6/6 笔交易入场价/出场价/盈亏与真实 5m **完全一致** (假设验证通过)。
2. **复权口径不一致时, 一致性检查会抓到** — 600000.SH (浦发) 除权敏感期 5m/1d 因子不同步 (差 ~4.7%), `adjust_mismatches=152` 报警。该防线是必要的, 不是摆设。
3. **002008 原 bug 场景已无法复现** — TDX 客户端登录同步后, 6.23~6.29 的 5m 缺口已从源头补齐; degrade 开/关双跑结果一致 (降级 0 股-天, 正确空转)。

## 实验设计

拿 5m 完整的股-天, 人为抹掉指定股-天的 5m bar, 双跑对账:
- A 基线: 真实 5m (degrade_5m 关)
- B 人为降级: 抹掉后 degrade_5m 开 → 1d 填充
- 按 (股票, 入场日) 匹配交易, 对比入场价/出场价/退出原因/盈亏

## 第一轮 (000001.SZ + 600000.SH + 002008.SZ, 2026-05~07 窗口)

| 指标 | 值 |
|---|---|
| 基线 / 降级交易 | 8 / 7 笔 (1 笔丢失) |
| 匹配交易入场价一致 | 4 / 7 |
| 出场价最大差 | 0.43 元 (成本止损), 0.43 元 (移动止盈), 0.33 元 (退市) |
| 总盈亏差 | +242.83 元 |
| **adjust_mismatches** | **152** ← 关键线索 |

偏差根因排查 (不是降级算法问题):
- **600000.SH: 5m=9.53 vs 1d=9.11, 全区间恒定 ~4.7% 差** — 浦发近期除权, 1d 缓存已吸收新前复权因子, 5m 侧 (缓存及 TDX 直拉) 尚未应用 → 两个数据层暂时不在同一复权基准上。填充的降级 bar 与周围 5m bar 价格跳变 4.7%, 触发/成交全乱 — **一致性检查正是为抓这个设计的, 工作正常**。
- 000001.SZ / 002008.SZ: 1d 缓存起点 2026-06-01, 5 月数据缺 (见"数据层遗留")。

## 第二轮 (剔除 600000.SH, 仅复权一致的 000001.SZ + 002008.SZ, 2026-06 窗口)

| 指标 | 值 |
|---|---|
| 基线 / 降级交易 | 6 / 6 笔 (零丢失) |
| 入场价一致 | **6 / 6** |
| 出场价最大差 | **0.0** (成本止损×1, 移动止盈×4, 阶梯止盈×1) |
| 总盈亏差 | **+0.00 元** |
| 降级报告影响金额 | [+0, +0] (本样本无模糊日, 正确) |

逐字节一致的原因: 入场 = 信号日 1d 收盘 = 真实收盘 (铁律); 出场 = 派生价
(stop_price / trail_line / ladder_price), 降级日 48 根 bar 的 high/low = 真实
1d 高低 → 触发集合与真 5m 相同, 派生成交价相同。同日顺序歧义 (模糊日) 本样本
未出现 — 出现时由 result.degradation 的 [悲观, 乐观] 区间量化, 不会静默。

## 002008 场景验证 (信号日 2026-06-23, 原缺口周)

- degrade_5m=False: 1 笔交易 (5m 缺口已被 TDX 补齐, 信号不再丢)
- degrade_5m=True: 同一笔交易, degradation 报告 0 降级股-天, 夏普 2.19 / 回撤 -0.48% 两跑一致
- 真实缺口填充路径已由第一轮覆盖: 6-08/6-17/6-22/6-24/6-29/7-07 等日三只股 5m 天然全缺 (×3), 均被 1d 真实数据填充并正常交易

## 数据层遗留 (建议另行处理)

> **2026-07-18 已修复 (commit: fix(cache) 复权因子漂移自愈)**: 第 1 条由"复权漂移
> 探针"(区间已覆盖时 12h 间隔比对 TDX 现值, 漂移自动全量重拉)+ force_invalidate
> (force_refresh 绕 F5 冷却) 解决; 探针对 intact=false 冷却期内的股同样生效。
> 第 2 条由 `tools/backfill_kline_cache.py` 全量回填 (1d 2015 起 / 5m 2024-06-27 起) 解决。
> 第 3 条仍是有效运维提示。

1. **F5 冷却 24h 使 force_refresh 对除权因子漂移无效** — 缓存的 F6 分红 shift 检测只在"增量扩展"时触发, 区间已覆盖时不检测; force_refresh 被 24h 冷却吞掉。除权敏感期 5m/1d 因子不同步最长可持续 24h+。建议: 1d 缓存加"因子 shift 检测"(同一股票同一日 close 与 TDX 现值比对), 或 force_refresh 绕冷却。
2. **000001.SZ / 002008.SZ 1d 缓存起点 2026-06-01** — 5 月及之前缺失, force_refresh 未回补 (告警 "缓存股票未预取")。降级遇到这些股-天会按"停牌"不填 (安全但丢信号)。
3. 除权敏感期运行降级时, 盯 `adjust_mismatches > 0` 的告警 — 当时的降级结果参考价值打折。
