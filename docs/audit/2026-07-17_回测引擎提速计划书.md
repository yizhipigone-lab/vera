## 2. Phase 1 — 零风险三项(已 A/B 验证)

### 2.1 涨停过滤全矩阵向量化

**现状**: [backtest/engine.py:1238-1251](../../backtest/engine.py#L1238) 逐列循环, 每列 pandas 算术 + `.loc` 赋值 + 每列调 `get_cached_info` 查 ST ([backtest/engine.py:1245](../../backtest/engine.py#L1245))。

**改法**:
- `close.values` 取出 → `prev = shift 一行`(numpy, 首行 NaN, 与 `close.shift(1)` 行为一致: NaN 比较恒 False → 首行不过滤) → `limit_up = cv >= prev*(1+ratio_vec)*0.997` 全矩阵一次算完(乘法顺序与现逐列逻辑完全一致, 无浮点重结合) → `entries & ~limit_up` 写回。
- ratio 向量(每列 0.10/0.20/0.05, 判定优先级复刻现逻辑: 688/300/301 先判, 否则查 ST)按"股票列集合"做缓存: 键 = `tuple(close.columns)` 的 hash, 存实例属性。同一批股票跑 78 个公式只查一次 ST 信息。
- 缓存有效期 = 进程内, 与 `get_cached_info` 自身缓存语义一致(ST 标记日频更新, 盘中变更概率极低; 重启进程即刷新)。
- 保留现有"移除 N 个涨停信号"的 info 日志(数字算法不变: `(entries.sum()-result.sum())`)。

**验收**:
- 新增测试: 随机数据对照旧实现(测试内联一份旧循环) `DataFrame.equals` 为 True; 主板/创业板/科创板/ST 四种 ratio 各至少 1 列; 首行 NaN 不过滤用例。
- A/B 已实测: 590ms → 3.0ms, equals=True。

### 2.2 买入信号预算表

**现状**: [backtest/loop/entry.py:48](../../backtest/loop/entry.py#L48) 每 bar `for ci in range(n_stocks)` 全列扫。

**改法**:
- `BacktestLoop.run()` 开头一次性 `sig_lists = np.split(np.nonzero(entry_np)[1], np.cumsum(entry_np.sum(axis=1))[:-1])`, 传给 `EntryEngine.run_bar` 替代逐列扫。
- 空 bar 边界已论证: `np.cumsum(sum(axis=1))[:-1]` 恒产 n_dates-1 个切点 → `np.split` 恒产 n_dates 段, 前/中/后空行都得空段(code-reviewer 复核确认)。
- 信号处理顺序保持列升序(`np.nonzero` 行主序, 同行内列升序, 与 `range(n_stocks)` 一致) → 同 bar 多信号买入顺序不变 → 成交/现金消耗顺序不变 → parity 不破。
- 与 F7 停牌计数兼容: skip 计数逻辑([backtest/loop/entry.py:52-58](../../backtest/loop/entry.py#L52))逐信号保留, 不变。
- 换股内层([backtest/loop/entry.py:61](../../backtest/loop/entry.py#L61))本阶段**不动**(持仓通常 ~30, 收益小, 字典维护有 swap-pop 同步风险)。

**验收**: A/B 已实测 25.5ms → 1.1ms; parity 测试全过。

## 3. Phase 2 — 交易记录向量化 + 测试收紧

### 3.1 `_build_trades` 向量化

**现状**: [backtest/engine.py:1163](../../backtest/engine.py#L1163) 逐行 dict + 每行 10 次 `round()`。

**改法**:
- raw 数组直接 `pd.DataFrame(raw, columns=[...])`; `stock_code` 用 `inv_col` 映射向量化; 日期用 `dates[idx_array]` 批量取。
- **舍入是最大坑, 两步走**(code-reviewer MEDIUM-1):
  1. 先写一致性探针测试: 构造 10 万随机值比对 `round(x, n)` vs `np.round(x, n)`(Python round 做正确十进制舍入, np.round 走二进制 half-even 快速路径, x.xx5 型值已知会发散, 如 `round(1.255,2)=1.25` vs `np.round(1.255,2)=1.26`)。
  2. 探针若不过(大概率): **fallback = 舍入列保留逐元素 Python round(列表推导), 只向量化 dict 构造/日期映射/索引映射**。收益大头在消除逐行 dict, 不在舍入本身, fallback 下 76ms→~20ms 仍成立。
- **顺序约束**(必须保持): 现逻辑 `entry_amount = round(ep*sh, 2)` 用的是**已 round 到 4 位后的 ep**(engine.py 内先 round4 再乘), 向量化必须保持"先 round4 → 再乘 → 再 round2"的同一数据流, 并用全等测试兜底。
- `hold_days` 向量化: `np.maximum(1, (xi-ei)//bpday)`, bpday==1 分支同理。

**验收**: 新旧输出 DataFrame 全等(值+顺序+列名); trades 为空/单行/含越界索引三类边界用例。

### 3.2 收紧性能回归测试

**现状**: [tests/test_loop_perf.py:52](../../tests/test_loop_perf.py#L52) `< 5.0s`、[:71](../../tests/test_loop_perf.py#L71) `< 3.0x`, 形同虚设。

**改法**: ratio 阈值 3.0 → 2.0(注释写明本地实测基线); 绝对阈值 5.0s → 1.0s。
**风险**: CI 机器慢可能误报 —— 本仓库测试全部本地跑, 无 CI, 风险可接受; 若未来上 CI 再放宽。

## 4. Phase 3 — 卖出侧向量化预筛(主菜, 单独排期)

### 4.1 思路

每 bar 对全部持仓用 numpy 一次算出"本 bar 是否**可能**有任何触发"(保守充分条件, 允许假阳性, **绝不允许假阴性**):
- 不可能触发的持仓: 只做 high 价跟踪(向量化 `np.maximum` 批量更新), 跳过 Context 构造和策略评估。
- 可能触发的持仓: 走现有完整 Python 路径(逐策略 evaluate, 逻辑一字不动)。

实测空跑率 84.7% → 预期卖出侧 350ms → ~120ms。

### 4.2 预筛条件推导(落地前必须逐策略写推导文档, 作为 PR 附件)

| 策略 | 保守触发必要条件 | 备注 |
|---|---|---|
| formula_sell(绝对优先) | `formula_exit_np[i-lag, ci]` 为真 | 直接向量化查表 |
| cost_stop | `pp <= threshold` 或跳空(open 相关) | 两分支取或 |
| ladder_tp | `hi_pp >= 下一未触发档位 profit` | 需从 ladder_done bitmask 向量化解出下一档, bitmask 最多 n_ladder 档, 可用查找表 |
| trailing | `peak_hi_profit >= activation` 且 `lo <= trail_line` | trail_line 依赖本 bar 更新后的 high_hi, 必须用"更新后"值算, 保守起见用 max(high_hi, 本bar high) |
| time_stop | `hold_days+1 >= max_hold` | 取边界保守值 |
| cond_time | 同 time_stop 模式 | |
| first_day | `hold_days <= n_bars` | |
| ATR | `atr_matrix[i, ci]` 非 NaN 且价格触及 | 矩阵已预算, 直接查 |

**任一策略条件拿不准 → 该策略对应持仓一律视为"可能触发"**(宁可慢不可错)。

### 4.3 状态一致性红线

1. **预筛标记一律按股票列 ci 键控, 严禁按 book 槽位索引**(code-reviewer MEDIUM-2): bar 内卖出触发 swap-pop 会重排槽位([backtest/loop/loop.py remove_swap_pop](../../backtest/loop/loop.py#L249)), 按槽位键控的标记会全部错位; 同一 ci 在 book 里唯一(同码先换股卖旧), 按 ci 键控天然免疫。
2. high_px/high_hi 必须对**所有**持仓每 bar 更新(含被预筛跳过的), 与现状逐仓更新字节级一致: 逐仓 `max(旧, 本bar close/high)` 与向量化 `np.maximum` 结果完全相同(无求和顺序问题), 安全。
3. ladder_done 只在触发路径内变, 预筛跳过路径不碰 → 安全。
4. T+1 当日买入持仓、停牌持仓的跳过逻辑保留在预筛之前, 顺序不变。

## 5. Phase 4 — 数据指纹缓存(随 parquet Phase 2 合并)

- 键: **全量内容 hash(blake2b)** of `(close.values, high, low, 相关参数)` → 缓存 ATR 矩阵、tradable_np、`close.values.astype(float64)` 结果。
- **严禁用 `id()` 或抽样 hash 做键**(code-reviewer HIGH-1): `id()` 在 GC 后可复用 → 误命中; 抽样 hash 有碰撞概率。误命中后果 = 用错 ATR 矩阵跑完整回测且**无任何报错**, parity 测试抓不到 —— 静默污染实盘决策依据, 性质最恶劣。
- 全量 hash 开销实测可忽略: 750×500 float64 = 3MB, blake2b 一次 ~2-3ms, 远小于被缓存的 ATR 计算。
- 参数扫描(同数据 N 组参数)场景生效; 单次调用无收益无损失。
- 与 `tools/gs_run_one.py` 批量路径对齐, 优先在批量入口生效; 与 5b55efb 的 `use_cache` 数据缓存分层(数据缓存管 K 线获取, 指纹缓存管计算产物), 不重复。
