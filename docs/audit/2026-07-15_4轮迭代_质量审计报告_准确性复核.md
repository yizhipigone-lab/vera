## 一、结论先说

原报告**发现问题的方向基本对，但严重级和运行时影响有明显失真**，不能直接拿原报告的“5 个 CRITICAL、6.8/10、不可交付”作为实施依据。

### 最关键的三项修正

1. **UX-C-1 是误报**：`Pipeline.run()` 失败后由 API 层重置 `pipeline_status.running`。实测连续两次模拟 TDX 连接失败，两次都返回 200 失败响应，随后状态均为 `running=False`；不存在“第二次永远 409、必须重启服务”。相关处理见 [server.py:331-350] 和 [server.py:361-369]。
2. **C-1 事实成立，但原修法无效、CRITICAL 定级过高**：`assert_single_path` 确实 0 次生产调用；然而单独调用 `assert_single_path(ENTRY_PATH)` 对回测路径和实盘路径都会通过，不能发现混用。函数真实语义见 [backtest/_entry_basis.py:110-128]，导入但未调用见 [backtest/loop/entry.py:19-24]。应降为 HIGH，并在有两个以上独立路径声明可比较时再接入真正守卫。
3. **C-2 是唯一经复核仍满足 CRITICAL 标准的问题，但原报告漏报了 Web/API 两层**：默认配置是 3.5% 激活、1% 回撤；引擎缺字段兜底、后端 API 缺字段兜底、页面初始值仍为 8%/5%。不同入口运行“默认策略”会得到不同退出行为和回测结果。

### 重新定级

| 原 CRITICAL | 事实判定 | 复核严重级 | 结论 |
|---|---|---:|---|
| C-1 路径守卫 0 生产调用 | 准确 | HIGH | 缺少运行期强制，但当前实盘路径尚未实现；原建议的单参数调用无保护价值 |
| C-2 移动止损止盈默认值漂移 | 准确且漏报范围 | **CRITICAL** | 直接改变回测退出行为，属于资金计算口径分裂 |
| T-CR-1 benchmark import 测试空转 | 准确 | HIGH | CI 假绿灯，事实实现当前仍正确，不是直接资金错误 |
| T-CR-2 dtype 测试自证自话 | 准确 | HIGH | 守卫与测试都无效，但当前数组容器已有 dtype 约束，不宜仅因测试弱升 CRITICAL |
| UX-C-1 TDX 失败后永久卡死 | **错误** | 无缺陷 | API 层已覆盖失败重置，实测连续失败可恢复 |

### 2.3 引用机器校验

原工具输出：20 个引用、20 个有效、0 个无效。**但这不是原报告所有代码引用。**

验证器正则只接受单段示例（如 `path/to/file.py` 第 42 行或第 42–51 行），定义见 [docs/audit/_verify_references.py:24-29]。以下复合引用不会被识别：

- `[backtest/engine.py:889-890, 1032-1033]`
- `[pipeline/pipeline.py:194-201, 215-219]`

此外，`.md`、`.yaml`、`.js`、`.html` 引用也完全不校验，测试明确把非 `.py` 引用排除，见 [tests/test_audit_references.py:76-80]。

因此原报告首页的二元主张（工具校验 + 主控 Read）只能判为 **PARTIAL**：被工具识别的 20 条全通过，但至少 2 条复合 Python 引用根本未进入工具；主控是否逐条做过语义比对是不可复现的黑盒，本复核既不能证明其完成，也不能证明其未完成。

## 五、唯一确认的 CRITICAL：默认策略口径分裂

### 5.1 证据链

权威默认配置：

- `config/default.yaml`：3.5% 激活、1% 回撤。
- 代码兜底同样是 3.5%/1%，见 [backtest/stop_config.py:109-148]。

仍使用 8%/5% 的入口：

- `BacktestEngine.run()` 缺字段兜底：[backtest/engine.py:882-912]
- `BacktestEngine.run_cached()` 缺字段兜底：[backtest/engine.py:1025-1054]
- 后端 `StrategyConfig.get()` 和 YAML 构造：`server.py:85-99`、`server.py:173-180`
- 页面初始表单：`web/index.html:368-374`
- 配置摘要缺字段兜底：[backtest/stop_config.py:87-92]

### 5.2 已复现影响

1. `StrategyConfig()` 生成的 Web 默认移动止损止盈配置是 0.08/0.05；`ConfigLoader.load_defaults()` 是 0.035/0.01。
2. 给 `run_cached()` 传 `{"trailing_stop": {"enabled": True}}`，捕获到核心引擎收到 0.08/0.05。
3. 因激活线和回撤线直接决定退出时机，该分裂会改变交易、权益曲线、夏普、Calmar、最大回撤和胜率，符合 CRITICAL 的“资金计算口径错误”标准。

### 5.3 安全修复边界

只修“字段缺失时的默认值”：

- **必须保留**用户明确传入的 8%/5%、3.9%/1.7% 或其他策略值。
- **不得覆盖**策略 YAML、`config/current.yaml`、浏览器 localStorage 已保存参数。
- **不得改变**`run_cached` 9 个旧位置参数、返回结构、T 日收盘买入口径、能力开关语义。
- 显式值 parity 必须锁定：改前显式传 8%/5%，改后核心引擎仍收到 8%/5%。
