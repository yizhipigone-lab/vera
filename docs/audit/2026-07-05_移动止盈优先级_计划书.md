## 1. 用户需求 (最终)

> "100 块, 到 103, 回撤到 101.97, 按 101.97 卖掉; 当天 Close 可能 96/97/98 (无所谓), Low 可能 94, 但盘中已触及 101.97 回撤线, 已清仓, 不按硬止损"

**两个独立改动**:
1. **优先级**: trailing 优先于 cost_stop (trailing_first 模式)
2. **语义**: trailing 从"Close 回撤 + Close 执行"改为"Low 触及回撤线 + 回撤线价执行"

## 2. 三档 priority 对照

| priority 值 | 同 bar 双触发时优先级 |
|---|---|
| `stop_first` (历史默认) | cost_stop > ladder_tp > trailing > cond_time > first_day > time |
| `ladder_tp_first` (当前默认) | ladder_tp > cost_stop > trailing > cond_time > first_day > time |
| **`trailing_first`** (新增) | ladder_tp > trailing > cost_stop > cond_time > first_day > time |

**注**: trailing 语义改造 (Low 触发 + 回撤线价 + 跳空保护) 是**全局改动**, 影响所有三个模式.

## 3. 决策点 (已确认, 含专家 D1-D4)

| 决策 | 选择 | 理由 |
|---|---|---|
| D1 pp>0/pp<=0 区分 | **保留 8/4 区分** (记录层面) | 用户能区分"锁利"和"止损"对账; "不区分"仅指优先级层面 |
| D2 触发模式开关 | **不加** (YAGNI) | 先 Low 触发, 有问题再加 |
| D3 HTML 默认勾选 | **HTML 跟 YAML 一致** (trailing_first) | 避免 B1 隐藏 bug |
| D4 engine_version 标签 | **本次不做** (后续单独迭代) | 工作量大, 与本任务正交 |

## 4. 必修 6 项 (专家审计, 开工前补进计划)

### 4.1 L1: peak_hi 时序修正

**问题**: engine.py:144-149 先更新 pos_high_hi 再算 peak_hi, 导致触发检测时 peak_hi 已含当根 high.

**修正**: trailing 触发检测用**更新前**的 peak_hi (上一根的峰值), 检测完再更新.

```python
# 修正: trailing 触发检测在 pos_high_hi 更新之前
peak_hi_for_trail = pos_high_hi[p] if pos_high_hi[p] > 0 else ep
# ... trailing 检测用 peak_hi_for_trail
# 然后 (现有逻辑) 更新 pos_high_hi
```

### 4.2 L3: trailing 跳空低开保护

**问题**: trailing 按回撤线价成交, 但开盘价已低于回撤线时 (如一字跌停) 高估成交价.

**修正**: 加 `min(sell_price, open)` 保护, 跟 cost_stop 同款.

```python
elif triggered in (4, 8):
    sell_price = peak_hi * (1.0 - trailing_drawdown)
    if open_np is not None:
        op = open_np[i, ci]
        if not np.isnan(op) and op < sell_price:
            sell_price = op   # 跳空低开保护
    actual_ret = (sell_price - ep) / ep if ep > 0.0 else 0.0
```

**注**: 专家 F5 的 `max(trail_line, low)` 写法错误 (高估), 采用 L3 的 `min(sell_price, open)`.
