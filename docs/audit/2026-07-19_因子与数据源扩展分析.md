## 0. 当前因子库的盲区(诚实)

目前腿 B 只用了 **3 个因子**:moneyflow(资金流)、top_inst(龙虎榜机构)、block_trade(大宗)。

**问题**:这 3 个都是"**短期资金博弈**"类因子。缺失三大类:
- **估值/基本面**(PE/PB/ROE/换手)— 中长期
- **杠杆/情绪**(融资融券/北向)— 市场情绪
- **文本/另类**(舆情/公告/研报)— 用户最初设想的 LLM 因子(一行没动)

## 二、TrendRadar(文本舆情因子)— 一行没动

之前评估过([计划书 §7.6.2](../plan/2026-07-19_AutoResearch量化增强_计划书.md)):newsnow 11+ 平台热榜 + AI 情绪/热度打分。

**能力**:
- 多平台热点聚合(知乎/微博/财联社/华尔街见闻等)
- AI 智能筛选(v6.5.0,自然语言兴趣 + min_score 阈值)
- MCP 17 工具:`analyze_sentiment`(情绪)、`analyze_topic_trend`(趋势)、跨平台关联
- Docker 本地部署,SQLite 自持

**可产出的因子**:
- **概念热度分**:某概念在热榜的曝光度/排名(时间序列)
- **舆情情绪分**:LLM 对个股相关新闻打情绪分(乐观/中性/悲观)
- **热点持续性**:概念热度持续天数(短线 vs 持续)

**落地路径**:
1. Docker 本地部署 TrendRadar
2. 配置 ai_interests.txt 关注金融/个股
3. 每日跑,SQLite 存热榜 + AI 分析
4. 输出因子表(日期 + 概念/代码 + 热度分 + 情绪分)
5. 注入 factor_score 评分

**难点**:
- TrendRadar 是"概念/热点"粒度,不是"个股"粒度 → 要做"概念→个股"映射(某概念热,其成分股加分)
- 历史回测:newsnow 是热榜快照,历史深度有限(只能前向积累,不能回溯)
- 适合**实盘增量**,历史回测覆盖差

## 三、公告 LLM 因子(cninfo + 本地 LLM)

**数据源**:巨潮 cninfo(证监会指定披露,免费 API)

**落地路径**:
1. 爬 cninfo 公告(按 ts_code + 日期)
2. 本地 LLM(Qwen2.5-14B + vLLM)读公告全文
3. LLM 提取:事件类型(增减持/回购/ST/业绩预告/定增)+ 利好利空强度
4. 输出因子表(日期 + 代码 + 事件类型 + 强度分)

**董秘优势**:你对公告解读是职业能力,LLM 打分 prompt 可以融入你的判断框架。

**难点**:
- 本地 LLM 部署(GPU,一次性投入)
- 公告全文爬取 + LLM 推理(计算密集)
- 前视防护:用 ann_date(公告日)对齐,不能用报告期(已在计划书 §7.7 写过)

## 四、其他数据源(iFinD/Choice/Wind)

用户之前说"没有商业终端账号,用开源组合"。这里分析它们能拿什么(供判断是否值得买):

| 源 | 能拿什么(tushare 拿不到的) | 成本 | 价值判断 |
|---|---|---|---|
| **iFinD(同花顺)** | 研报全文 + 一致预期(盈利预测明细)+ 行业景气 + 产业链文本 | 几万/年 | 研报因子要它 |
| **Choice(东财)** | 研报全文 + 公告全文 + 新闻全文 | 几万/年 | 文本全 |
| **Wind** | 全文本 + 机构持仓明细 + 调研纪要 + 实时 | 几十万/年 | 最全最贵 |
| **AkShare(开源)** | 东财/同花顺爬的补充(部分全文) | 免费 | 备选 |

**核心判断**:**tushare 拿不到的是"全文文本"**(研报全文/公告全文/调研纪要)。结构化数据 tushare 基本够。要做"LLM 读全文做深度因子",才需 iFinD/Choice/Wind。

**建议**:**先不买**。tushare 结构化因子(补 daily_basic/margin/hk_hold)+ TrendRadar(舆情)+ cninfo 公告(免费),已经覆盖大部分。只有"研报一致预期"这类深度文本因子,才真需要 iFinD。
