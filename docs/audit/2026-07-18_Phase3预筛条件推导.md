# Phase 3 预筛条件逐策略推导 — 为什么零假阴性

> 日期: 2026-07-18
> 目的: 卖出侧向量化预筛(跳过 84.7% 空跑)的安全性论证。预筛返回 False(不可能触发)时,
> 必须**数学上保证**该持仓当 bar 经完整路径(absolutes + dispatcher.evaluate)也零触发。
> 允许假阳性(多评估, 只是慢), 绝不允许假阴性(漏评估, 结果错)。

## 0. 预筛放置点与状态一致性

预筛内联在 `_sell_bar` 每持仓循环内, 位于"衍生量计算"之后、`book.get/Bar/Context 构造`之前。

关键事实: 当前代码中, `high_px`/`high_hi` 的更新发生在衍生量计算段(预筛之前),
与是否评估策略无关 — 所以跳过评估**不会改变任何跨 bar 状态**:
- `ladder_done` 只在 ladder 策略判定 `new_mask != prev_mask`(即真触发)时才 mutate — 零触发时写回原值, 与不写等价
- swap-pop 只发生在真执行时 — 零触发时无结构变化
- 预筛在同一迭代内就地判定, 不跨迭代持有标记 → 与槽位重排天然无关(无 MED-2 所述槽位错位问题)

## 1. 逐策略必要条件推导

记: ep=entry_px(>0), hi/lo=当根 High/Low, hi_pp=(hi-ep)/ep, lo_pp=(lo-ep)/ep,
peak=更新后 high_hi(>0 时, 否则 ep), hold=i-entry_idx。

| 策略 | 触发条件(代码证据) | 预筛必要条件 | 无假阴性论证 |
|---|---|---|---|
| formula_sell | `signal[i-lag, ci]` 为真 ([absolute.py:44](../../backtest/loop/absolute.py#L44)) | `i>=lag 且 signal[i-lag,ci]` 为真 | 条件即触发本身, 恒等 |
| cost_stop | `lo_pp <= threshold` ([cost_stop.py:28](../../backtest/loop/strategies/cost_stop.py#L28)) | `lo_pp <= threshold` | 条件即触发本身, 恒等。跳空保护只改执行价不改触发 |
| ladder_tp | `new_mask != ladder_done`, 位 li 置位 ⟺ `hi_pp >= profits[li]` ([ladder_tp.py:40-44](../../backtest/loop/strategies/ladder_tp.py#L40)) | `hi_pp >= min{profits[li]: 位 li 未置位}` | 若 hi_pp 小于所有未触发档位利润, 则新掩码无任何新置位 → new==prev → 必不触发 |
| trailing | `peak_hi_profit >= activation 且 lo <= peak*(1-drawdown)` ([trailing.py:29-32](../../backtest/loop/strategies/trailing.py#L29)) | 同左两条件同时成立 | 两条件即触发本身, 恒等。peak 用更新后 high_hi, 与 ctx 一致 |
| time_stop | `hold_days >= max_hold_days` ([time_stop.py:23](../../backtest/loop/strategies/time_stop.py#L23)) | `hold >= mhd` | 恒等 |
| cond_time | `hold >= days 且 hi_pp >= profit` ([cond_time.py:25](../../backtest/loop/strategies/cond_time.py#L25)) | 同左 | 恒等 |
| first_day | `cur_day==entry_day+1 且 bar%bpday==bpday-1 且 day1_return<target` ([first_day.py:28-38](../../backtest/loop/strategies/first_day.py#L28)) | 仅取前两时间条件(放宽) | 时间不满足则必不触发; 时间满足时一律放行(价格条件留给全路径判) — 保守方向正确 |
| atr_stop | `atr>0 且 lo <= peak - mult*atr` ([atr_stop.py:41-44](../../backtest/loop/strategies/atr_stop.py#L41)) | 同左 | 恒等(atr 矩阵越界/NaN 时返回不可能 → 与策略返回 [] 一致) |

## 2. 组合规则

`could_trigger = OR(各已启用策略的必要条件)`。策略未启用(capability gating)则跳过。
**拿不准一律返回 True(走全路径)** — 宁可慢, 不可错。

## 3. 验证设计

1. **55 组 parity**: 新旧字节级一致(预筛属内部实现, parity 必须全绿)。
2. **假阴性监控(属性测试)**: 随机 100 组场景, 对预筛判 False 的每个 (持仓,bar),
   强制走全路径 evaluate + absolutes, 断言结果为空 — 直接证明"跳过=零触发"。
3. **预筛强制关闭对照**: 环境变量/参数强制 could_trigger=True 跑一遍, 与开启版字节一致。

## 4. 性能预期

实测空跑率 84.7%: 被跳过的持仓省下 book.get + Bar/Context 构造 + 4-7 次策略 check
(~15µs), 预筛本身 ~1-2µs(纯标量比较链, 短路求值)。预期卖出侧 ≥2.5x。
