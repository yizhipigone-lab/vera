def _capture_core_trailing(monkeypatch, n_bars):
    captured = {}
    parameter_names = list(
        inspect.signature(engine_module._simulate_core_v3).parameters
    )
    activation_index = parameter_names.index("trailing_activation")
    drawdown_index = parameter_names.index("trailing_drawdown")
    assert activation_index == 11
    assert drawdown_index == 12

    def fake_core(*args, **kwargs):
        # 兼容签名一旦改变，上面的名称索引断言会先给出明确失败。
        assert len(args) > drawdown_index
        captured["activation"] = args[activation_index]
        captured["drawdown"] = args[drawdown_index]
        return (
            np.full(n_bars, 100_000.0, dtype=np.float64),
            np.empty((0, 9), dtype=np.float64),
        )

    monkeypatch.setattr(engine_module, "_simulate_core_v3", fake_core)
    return captured

class InputValueParser(HTMLParser):

Every dimension must cite tests, reproduction, coverage, or code evidence. Do not add subjective bonus points.

- [ ] **Step 2: 写实施复核报告**

The report must include:

- 修复前/后默认值链路对比。
- 明确值 8%/5%、0%、None、缺字段四类契约结果。
- 全量测试、skip、coverage、lint/type-check 状态。
- code-reviewer 发现及处置。
- 未覆盖范围。
- 真实系统评分。

- [ ] **Step 3: 依据真实评分决定下一轮**

Expected decision:

- 本 CRITICAL 修复完成后，系统分数会上升，但在 JSON 序列化、假测试、端到端稳定性、共享常量和 70% 覆盖率尚未处理时，通常仍不会真实达到 9.5。
- 若低于 9.5：从剩余 HIGH 中选一个独立根因，重新执行“设计 → 用户批准 → 详细计划 → 用户批准 → TDD 实施 → 全量验证”。
- 不把多个独立子系统塞入一次大改。

- [ ] **Step 4: 仅在所有证据满足时宣布 9.5–9.8**

Minimum evidence for final target:

- 0 CRITICAL / 0 HIGH 未处理发现。
- 全量测试 0 failed。
- 稳定的非外部依赖关键集成测试通过。
- 全仓覆盖率至少 80%，或用户批准且有书面理由的等价风险覆盖证据。
- lint/type-check 可用并通过，或项目正式配置明确替代工具。
- 真实 Web 默认值、API 默认值、引擎默认值和配置文件一致。
- 无旧接口、旧显式配置、T 日收盘买入、优先级或 typed 返回回归。

**Checkpoint:** 该循环必须由证据停止，不能因为“已经循环很多轮”而停止。
